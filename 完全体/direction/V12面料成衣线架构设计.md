## 二、结论先行（四个核心判断）

### 判断 1：一致性地基没了 → 两段式虚拟打样（本线灵魂）

V9 系 8 图同人同衣，靠每张生图卡槽2挂同一张商品白底图（连线即锁）。面料线成衣不存在，**没有锚图 = 8 张图各自发明领口/口袋/下摆 = 跨图一致性必然失守**（纯文字锁不住几何细节，V11.1 蓝花事件实锤）。

**方案对比**：
- 方案A 一段式直出：省打样费，但无成衣锚，商用不可行。
- **方案B 两段式虚拟打样（Joe 选定）**：阶段一先"打样"出 GS1 套装白底正面 + GS2 背面 + GS3 模特定妆照（2-3张，便宜），人工核可后回填当商品图/模特图，**完整复用 V9.2 成熟骨架与全部护栏**。样衣白底图本身也是可交付展示素材。贴合供应链心智：先打样、再拍大片。
- 方案C 直出+事后对齐：最贵且误差累积（对齐锚是 AI 图的 AI 图），不选。

### 判断 2：M 矩阵重组 = 「F1 + D1 + CS」新建，「SP1」改造，其余复用

原 M1/M5/M6（读成衣的分析官）失效。按核心理念判断1"分析官按输出物分组"：
- **F1 面料物理锁定官**（被动分析——面料是唯一实物，必须忠实）
- **D1 奢牌版型设计官**（主动设计——全流程唯一"创作型"节点，内置四品牌版型语言库）
- **CS 打样编译器**（小型C，产3组打样提示词）
- **SP1 氛围导演官**（A3 裂变：从"分析用户场景图"变为"按品类调性主动策划"）
- 阶段二 A1/A2/C/P×8/G×8/QA/R/RG/X 全部复用。

### 判断 3：面料"1:1还原"重定义为按景别分级 + 真源直连 + 四级贯通

像素账（铁律20）：2x2罗纹纹目约2mm；3:4/2K 全身位约 1.1px/mm → 纹目仅 2px，物理上画不出也不该画出（真实相机同距离同样拍不出）。分级标准见第五节。锁定三件套：面料原片直连每张G卡细节槽（物理锁）；克重/成分/光泽/垂坠四级贯通（REQ→F1强制覆盖→C正向短语+负向词常驻→QA专项+硬失败）；枚举令沿用（铁律19，禁"keep the fabric texture"空话）。

### 判断 4："没想法"托管 = 数据层映射表 + 三档降级

品类→奢牌调性映射表写在 SP1 提示词的数据字段区与分镜表模板（数据层），规则引擎不碰（铁律21）。托管三档：没想法=全托管；一句话意向=意向优先映射表补全；给参考图=退回V9老路。

## 三、阶段一设计（虚拟打样）

### 3.1 输入协议

| 输入 | 必/选 | 用途 |
|---|---|---|
| IN-F1 面料细节图 | 必填 | 颜色+纹理唯一真源；同时直连阶段二每张G卡细节槽（判断11真源优先） |
| IN-F2 面料色卡图 | 选填 | F1 读印刷参数（克重/成分/幅宽）与配色板 |
| IN-M 模特图 | 选填 | 不给则 GS3 自动产定妆照 |
| 需求卡① 品类与输出 | 必填 | 品类（家居服/老钱针织/极简运动/绅装家居…）、性别年龄、输出比例数量（默认 3:4） |
| 需求卡② 硬指令字段集 | 有默认值 | 见 3.2 |

### 3.2 需求卡② 字段（全部有默认值，空着能跑）

| 字段 | 默认值 | 说明 |
|---|---|---|
| 克重 | 空（F1推断+UNCERTAIN） | 填了强制覆盖视觉推断 → 垂坠/厚薄 |
| 成分 | 空（同上） | → 光泽/绒感/弹力 |
| 面料品类 | 空（F1判定，拿不准UNCERTAIN问人） | 用户可显式指定（如"华夫格"） |
| 上身松紧 | relaxed 松弛感 | 三态口径沿用（修身/压缩/宽松） |
| 套装构成 | 上衣+裤子 | 可改连体/三件 |
| 奢牌风格定向 | 自动按品类从四品牌池选 | Loro Piana/Brunello Cucinelli ｜ Zimmerli/Derek Rose ｜ Hermès ｜ Ralph Lauren Purple Label |
| 场景姿势意向 | 没想法（全托管） | 也接受一句话意向（半托管） |

### 3.3 F1 面料物理锁定官（输出：面料DNA锁卡）

输入：IN-F1 + IN-F2（若有）+ 需求卡②。中文指令+英文值，Agent八要素。8字段：
1. 面料品类与组织结构（如 2x2 rib knit；需求卡填了则强制覆盖）
2. 主色（英文色名+HEX目测值，以细节图为准）
3. 纹理几何（罗纹间距/织向/纱线粗细档）
4. 光泽档（matte / dry-touch / subtle sheen / soft luster / silk luster 五档）
5. 表面手感（brushed / smooth / crisp / slub …）
6. 物理行为（垂坠/挺括/弹力——克重成分字段强制覆盖视觉推断，空则推断+UNCERTAIN）
7. 适用品类建议
8. 风险提示（如同色提花难还原、色卡与细节图色差）

下游：D1、CS、阶段二C（锁卡直通，不经A1转手——防二次压缩丢信息，V11.1教训）。

### 3.4 D1 奢牌版型设计官（输出：成衣设计锁卡，人类可读可改）

输入：F1锁卡 + 需求卡①②。提示词内置**四品牌版型语言库**（数据层字段区）：LP/BC=老钱风顶级羊绒针织、松弛感剪裁、大地色系；Zimmerli/Derek Rose=奢华家居服、精致包边、正装睡衣剪裁；Hermès=极简考究线条；RLPL=修身雅致贵族针织绅装。7字段：
1. 对标基因（选用哪家哪种版型语言+一句理由，给人看）
2. 单品清单（每件：品名/廓形/领袖摆细节/工艺如包边罗纹缝/闭合方式）
3. 配色运用（默认同面料同色套装——家居服惯例）
4. 尺寸比例感（oversized/regular/slim）
5. 造型层次（单穿/叠穿）
6. 模特设定（性别/年龄段/气质/发色发型；需求卡①的性别年龄有填则强制沿用，D1只补齐其余；用户给模特图则跳过此字段）
7. UNCERTAIN 风险

**D1 合规铁则**：只借版型语言与调性，禁止设计任何品牌 logo、文字、标志性专属花纹（知识产权+铁律13双重理由）。

### 3.5 CS 打样编译器与 GS 生图卡

CS（纯英文）读 F1+D1 锁卡 → 3组打样提示词（各含负向词：no text, no logo, no extra garment, no mannequin visible…）。

| 卡 | 内容 | 参考图槽位（连线序=声明序，铁律1/8） | 分辨率 |
|---|---|---|---|
| GS1 | 套装白底正面（隐形人台质感产品图） | 槽1=面料细节图 | 2K（事实源质量=全线上限）★待实测 |
| GS2 | 套装白底背面 | 槽1=面料细节图 | 1K |
| GS3 | 模特定妆照（穿样衣、素背景全身） | 槽1=GS1成品 ＋ 槽2=面料细节图（用户给模特图则加槽3） | 1K |

GS3 挂 GS1 成品 = 生图卡输出接生图卡输入（X 节点已验证的图→图链路）。运行顺序：GS1/GS2 并行 → GS3。

### 3.6 人工检查点（组S后置，核可清单）

Joe 核可样衣后才进阶段二。核可清单（写进使用说明卡）：
- 面料对不对：纹理类型、颜色、光泽 vs 面料原片
- 设计合理性：缝线走向、闭合逻辑、口袋左右对称（**AI伪影若被核可，会被A1当事实锁定全程复制——垃圾进垃圾出**）
- 版型是否符合选定的奢牌基因
- 不满意：改需求卡/D1锁卡 → 单卡重跑该 GS（便宜）

无论托管哪一档，SP1 输出结构不变：场景DNA + **一句可逐字复用的场景句**（R5双锁沿用）+ 8分镜表。

### 4.3 分镜表（展示大片定位）

| 镜 | 定位 | 备注 |
|---|---|---|
| G1 | 全身正面版型位 | 奠定套装全貌 |
| G2 | 背面全身 | 骨架必含背面结构位 |
| G3 | 半身上装质感位 | 织纹类型可辨级 |
| G4 | **面料贴肤特写位**（袖口/领口/胸前） | 2K强制，纹理1:1核验位 |
| G5–G7 | 场景生活位 | 姿态库轮换、景别递进 |
| G8 | 氛围大片位 | 环境大占比、人物中景，封面候选 |

分辨率默认：G卡全2K（奢牌质感线，对齐V11惯例；预算紧可降G1-G3/G5-G7为1K）★待实测定稿。

### 4.4 C 编译器新增面料规则段（骨架规则10条全保留）

- 纹理跨景别一致性法：近景纹理=面料原片1:1；全身=该距离应有观感；禁把罗纹画成平纹
- 光泽法：按F1光泽档注入正向短语；哑光面料常驻负向 `glossy, satin sheen, plastic-like fabric`
- 垂坠法：克重驱动短语（如 220g modal rib → soft heavy drape, close skin contact）
- 材质降级防线：常驻负向 `cheap fabric look, polyester shine, flattened texture`
- 枚举令沿用（铁律19）：复述面料锁卡逐项点名（组织结构/纱线档/光泽/手感）

### 4.5 QA 新增维度（认图规则/评分口径/返修简报结构全沿用）

- **面料还原（最高优先级）**，按景别分级：特写位=纹理结构/纱线粗细/色准逐项对面料原片；半身位=织纹类型可辨+光泽档正确；全身位=垂坠形态/光泽/色准远观正确。色偏可感=硬失败；织纹类型画错（罗纹→平纹）=硬失败
- **成衣与样衣一致**：GS1 为唯一事实源（原商品保真维度直接对齐）
- **合规**：出现品牌logo/文字/专属花纹=硬失败

## 五、像素账（铁律20，接单时已算）

| 景别 | 分辨率 | 像素密度 | 2mm罗纹纹目 | 还原标准 |
|---|---|---|---|---|
| 全身位 | 2K | ~1.1px/mm | ~2px | 远观质感：垂坠/光泽/色准 |
| 半身位 | 2K | ~2.5px/mm | ~5px | 织纹类型可辨 |
| 特写位 | 2K | ~5px/mm | ~10px | 纹理结构1:1核验 |

结论：特写位必须2K+面料原片挂细节槽；全身位不数织目（真实相机也拍不出），QA按分级验收。
