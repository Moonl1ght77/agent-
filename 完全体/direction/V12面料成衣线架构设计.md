## 一、业务定位与问卷拍板记录

**业务痛点**：供应链源头只有面料，没有成衣。输入一张面料细节图+品类文字，系统自动设计一套高奢成衣（默认上衣+裤子）并渲染成模特试穿展示大片。

**Joe 拍板（2026-07-10）**：
1. **出图用途 = 面料商展示**（放首页等位置做"外观提升"，非面料详情页、非电商主图）→ 分镜表按展示大片排、QA 最高优先级 = 面料还原与高奢调性。
2. **输入协议 = 单图必填+其余选填**：面料细节图必填；色卡图、克重/成分字段选填——有则物理精度大增，无则视觉推断+UNCERTAIN 降级。兼容任何来源的面料（乐顺三件套只是当前实例）。
3. **模特 = 可选上传，缺省自动生成**定妆照并全程锁定。
4. **核心架构 = 方案B 两段式虚拟打样**（见判断1）。
5. **双模式（2026-07-10 Joe补充）**：模式A=图驱动（面料照片为纹理真源）；**模式B=类目文驱动**——无面料图，用户选面料类目+颜色，F1按内置类目物理库推物理行为，出图走纯色极简 quiet luxury（"越简单越高级"），版型照用四大牌语言库。模式B既独立可跑，也是模式A纹理还原失败后的人工降级出路（不自动降级）。
6. **第二次增补（2026-07-10 Joe）**：①输出从8分镜减为**4分镜**（分镜数N是骨架插槽，运行组结构不变）；②类目库升级为**三轴详表**（纤维成分×织造结构×产品用途，见3.2.1）——用途轴与需求卡①品类字段合并，不重复设字段。

**首批实测素材**：`E:\微信小程序开发总文件\02-assets\client-jiajufu\乐顺小程序`——5款针织面料，每款含面料细节图+色卡图（印有款号/幅宽/克重/成分+全部配色）+产品信息.xlsx（成分/克重/适用场景）。

## 二、结论先行（四个核心判断）

### 判断 1：一致性地基没了 → 两段式虚拟打样（本线灵魂）

V9 系 8 图同人同衣，靠每张生图卡槽2挂同一张商品白底图（连线即锁）。面料线成衣不存在，**没有锚图 = N 张图各自发明领口/口袋/下摆 = 跨图一致性必然失守**（纯文字锁不住几何细节，V11.1 蓝花事件实锤）。

**方案对比**：
- 方案A 一段式直出：省打样费，但无成衣锚，商用不可行。
- **方案B 两段式虚拟打样（Joe 选定）**：阶段一先"打样"出 GS1 套装白底正面 + GS2 背面 + GS3 模特定妆照（2-3张，便宜），人工核可后回填当商品图/模特图，**完整复用 V9.2 成熟骨架与全部护栏**。样衣白底图本身也是可交付展示素材。贴合供应链心智：先打样、再拍大片。
- 方案C 直出+事后对齐：最贵且误差累积（对齐锚是 AI 图的 AI 图），不选。

### 判断 2：M 矩阵重组 = 「F1 + D1 + CS」新建，「SP1」改造，其余复用

原 M1/M5/M6（读成衣的分析官）失效。按核心理念判断1"分析官按输出物分组"：
- **F1 面料物理锁定官**（被动分析——面料是唯一实物，必须忠实）
- **D1 奢牌版型设计官**（主动设计——全流程唯一"创作型"节点，内置四品牌版型语言库）
- **CS 打样编译器**（小型C，产3组打样提示词）
- **SP1 氛围导演官**（A3 裂变：从"分析用户场景图"变为"按品类调性主动策划"）
- 阶段二 A1/A2/C/P×4/G×4/QA/R/RG/X 全部复用（分镜数N=4，Joe定；N是骨架插槽，运行组结构不变）。

### 判断 3：面料"1:1还原"重定义为按景别分级 + 真源直连 + 四级贯通

像素账（铁律20）：2x2罗纹纹目约2mm；3:4/2K 全身位约 1.1px/mm → 纹目仅 2px，物理上画不出也不该画出（真实相机同距离同样拍不出）。分级标准见第五节。锁定三件套：面料原片直连每张G卡细节槽（物理锁）；克重/成分/光泽/垂坠四级贯通（REQ→F1强制覆盖→C正向短语+负向词常驻→QA专项+硬失败）；枚举令沿用（铁律19，禁"keep the fabric texture"空话）。

### 判断 4："没想法"托管 = 数据层映射表 + 三档降级

品类→奢牌调性映射表写在 SP1 提示词的数据字段区与分镜表模板（数据层），规则引擎不碰（铁律21）。托管三档：没想法=全托管；一句话意向=意向优先映射表补全；给参考图=退回V9老路。

### 判断 5：双模式共用单份规则引擎——真源模式字段切换分支

模式A/B 不做两套 C/QA（变体禁止复制规则引擎，V10教训）：F1 锁卡新增**字段0"真源模式"**（photo-locked / category-derived），C 与 QA 内置双分支按该字段切换（先例=R5 场景类型双模式）。结构差异（模式B的GS/G卡无面料参考图→挂图数与前缀版本不同）用 `--variant` 机制落地（V10先例）：`build --spec workflow_spec_v12.json --variant b`。

模式B的扬长避短逻辑：生图模型最不稳的是复刻特定纹理，最稳的是纯色好垂坠——模式B主动放弃纹理复刻，QA 从"纹理1:1"换轨为"纯色纯净度+物理行为可信+跨图色一致"。

### 3.2.1 面料类目物理库（模式B真源，数据层，放F1提示词字段区；三轴详表=Joe 2026-07-10定稿）

F1 物理推导 = **轴一（纤维→光泽/手感/垂坠基调）× 轴二（结构→弹性/表面/形态）× 克重档修正**。轴三（用途）=需求卡①品类，不进物理库。

**轴一：纤维成分**

| 一级 | 二级细分 | 物理默认档（光泽/手感/垂坠） |
|---|---|---|
| 天然纤维·棉 | 棉 | 哑光 / 干爽柔软 / 中垂坠 |
| 天然纤维·麻 | 亚麻、苎麻、汉麻 | 哑光带干涩肌理 / 挺爽 / 低垂坠天然褶 |
| 天然纤维·毛 | 羊毛、羊绒、羊驼毛、马海毛 | 哑光微绒（马海毛带毛羽光晕）/ 暖糯 / 中垂坠；羊绒=micro-fuzz halo |
| 天然纤维·蚕丝 | 桑蚕丝、柞蚕丝 | soft luster流动光 / 凉滑 / 高垂坠 |
| 再生纤维素纤维 | 粘胶、莫代尔、莱赛尔/天丝、竹纤维、醋酸、铜氨 | 柔光subtle sheen（醋酸/铜氨偏丝光）/ 滑糯贴肤 / 高垂坠 |
| 合成纤维 | 涤纶、锦纶、氨纶、腈纶、丙纶 | 微光~亮光（负向常驻防廉价涤纶光）/ 爽滑 / 中垂坠；氨纶只做弹力修正项不单独成衣 |
| 混纺面料 | 棉涤、棉锦、棉氨、粘胶氨纶、莫代尔氨纶、羊毛涤纶、锦纶氨纶 | 按主纤维取主导档+副纤维修正（含氨纶=弹力升档）；比例未知则UNCERTAIN |

**轴二：织造结构**

| 一级 | 二级细分 | 物理默认档（弹性/表面/形态） |
|---|---|---|
| 针织 | 单面布、双面布、罗纹、珠地、毛圈、提花针织、经编、纬编 | 有弹随体、边缘罗纹收口；罗纹=纵向棱线、珠地=蜂窝网眼、毛圈=绒圈感。适合家居服/内衣/T恤/运动 |
| 梭织 | 平纹、斜纹、缎纹、提花、牛津纺、府绸、雪纺、色丁 | 无弹挺括（加氨纶除外）、有裁片结构线；缎纹/色丁=satin sheen、雪纺=轻透飘、斜纹=可见斜纹路。适合衬衫/裤装/外套/裙装 |
| 非织造 | 无纺布、熔喷布、水刺布、针刺布 | **范围外**：医疗/清洁/包装/工业用，不属成衣线，选中则提示不适用（铁律14不硬编） |

**范围声明**：用途轴中"家纺面料、鞋材和箱包面料"同属范围外——V12是成衣线，选中提示不适用。功能性诉求（速干/冰感等）走用途轴品类（运动/瑜伽/泳装）+二级细分表达。二级细分缺失时按一级默认档跑并标注UNCERTAIN；首轮实测后校准各档措辞。

### 3.3 F1 面料物理锁定官（输出：面料DNA锁卡）

输入：IN-F1 + IN-F2（若有）+ 需求卡②。中文指令+英文值，Agent八要素。字段0+8字段：
0. **真源模式**：photo-locked（有面料图）/ category-derived（模式B：按3.2.1类目物理库+需求卡字段查表推导，全部字段照常输出但标注来源=类目库）——下游C/QA按此字段切换分支
1. 面料品类与组织结构（如 2x2 rib knit；需求卡填了则强制覆盖）
2. 主色（英文色名+HEX目测值，以细节图为准）
3. 纹理几何（罗纹间距/织向/纱线粗细档）
4. 光泽档（matte / dry-touch / subtle sheen / soft luster / silk luster 五档）
5. 表面手感（brushed / smooth / crisp / slub …）
6. 物理行为（垂坠/挺括/弹力——克重成分字段强制覆盖视觉推断，空则推断+UNCERTAIN）
7. 适用品类建议
8. 风险提示（如同色提花难还原、色卡与细节图色差）

下游：D1、CS、阶段二C（锁卡直通，不经A1转手——防二次压缩丢信息，V11.1教训）。

### 3.4 D1 奢牌版型设计官（输出：成衣设计锁卡，人类可读可改）

输入：F1锁卡 + 需求卡①②。提示词内置**四品牌版型语言库**（数据层字段区）：LP/BC=老钱风顶级羊绒针织、松弛感剪裁、大地色系；Zimmerli/Derek Rose=奢华家居服、精致包边、正装睡衣剪裁；Hermès=极简考究线条；RLPL=修身雅致贵族针织绅装。7字段：
1. 对标基因（选用哪家哪种版型语言+一句理由，给人看）
2. 单品清单（每件：品名/廓形/领袖摆细节/工艺如包边罗纹缝/闭合方式）
3. 配色运用（默认同面料同色套装——家居服惯例）
4. 尺寸比例感（oversized/regular/slim）
5. 造型层次（单穿/叠穿）
6. 模特设定（性别/年龄段/气质/发色发型；需求卡①的性别年龄有填则强制沿用，D1只补齐其余；用户给模特图则跳过此字段）
7. UNCERTAIN 风险

**D1 合规铁则**：只借版型语言与调性，禁止设计任何品牌 logo、文字、标志性专属花纹（知识产权+铁律13双重理由）。

**D1 模式B追加约束**：纯色、无图案无提花无印花、工艺细节做减法（quiet luxury，"越简单越高级"）——设计亮点只允许落在廓形、剪裁、领袖摆收口方式上。

### 3.5 CS 打样编译器与 GS 生图卡

CS（纯英文）读 F1+D1 锁卡 → 3组打样提示词（各含负向词：no text, no logo, no extra garment, no mannequin visible…）。

| 卡 | 内容 | 参考图槽位（连线序=声明序，铁律1/8） | 分辨率 |
|---|---|---|---|
| GS1 | 套装白底正面（隐形人台质感产品图） | 槽1=面料细节图 | 2K（事实源质量=全线上限）★待实测 |
| GS2 | 套装白底背面 | 槽1=面料细节图 | 1K |
| GS3 | 模特定妆照（穿样衣、素背景全身） | 槽1=GS1成品 ＋ 槽2=面料细节图（用户给模特图则加槽3） | 1K |

GS3 挂 GS1 成品 = 生图卡输出接生图卡输入（X 节点已验证的图→图链路）。运行顺序：GS1/GS2 并行 → GS3。

**模式B（--variant b）槽位差异**：GS1/GS2 无参考图（纯文生图，0图前缀版本）；GS3 只挂槽1=GS1成品（1图前缀）。挂几张声明几张（铁律8），前缀版本随变体切换由构建器保证。

### 3.6 人工检查点（组S后置，核可清单）

Joe 核可样衣后才进阶段二。核可清单（写进使用说明卡）：
- 面料对不对：纹理类型、颜色、光泽 vs 面料原片

- 版型是否符合选定的奢牌基因
- 不满意：改需求卡/D1锁卡 → 单卡重跑该 GS（便宜）

### 4.4 C 编译器新增面料规则段（骨架规则10条全保留）

- 纹理跨景别一致性法（**仅photo-locked分支**）：近景纹理=面料原片1:1；全身=该距离应有观感；禁把罗纹画成平纹
- 纯色纯净法（**仅category-derived分支**）：solid color, clean uninterrupted surface；常驻负向 `printed pattern, jacquard, embroidery, texture noise, visible logo`
- 光泽法：按F1光泽档注入正向短语；哑光面料常驻负向 `glossy, satin sheen, plastic-like fabric`（双分支共用）
- 垂坠法：克重驱动短语（如 220g modal rib → soft heavy drape, close skin contact）（双分支共用；B分支取克重档默认值）
- 材质降级防线：常驻负向 `cheap fabric look, polyester shine, flattened texture`（双分支共用）
- 枚举令沿用（铁律19）：复述面料锁卡逐项点名（组织结构/纱线档/光泽/手感）
- 分支切换依据=F1锁卡字段0，C内双分支单份引擎（判断5）

### 4.5 QA 新增维度（认图规则/评分口径/返修简报结构全沿用）

- **面料还原（最高优先级，按F1字段0切分支）**：
  - photo-locked：按景别分级——特写位=纹理结构/纱线粗细/色准逐项对面料原片；半身位=织纹类型可辨+光泽档正确；全身位=垂坠形态/光泽/色准远观正确。色偏可感=硬失败；织纹类型画错（罗纹→平纹）=硬失败
  - category-derived：纯色纯净度（出现任何图案/提花/印花=硬失败）+ 物理行为可信（垂坠/光泽符合类目档）+ 跨图色一致（4张同色，色漂移=硬失败）
- **成衣与样衣一致**：GS1 为唯一事实源（原商品保真维度直接对齐，双分支共用）
- **合规**：出现品牌logo/文字/专属花纹=硬失败（双分支共用）
