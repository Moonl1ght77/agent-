## 二、画布临时卡（蓝图里没有，靠本文件重建）

### ★ GSCENE 换场景位 —— 今天唯一验证成功、Joe 认可的产出
提示词已存档：`blueprints/prompts/GSCENE_换场景_v13.txt`

| 卡 | 类型 | 作用 |
|---|---|---|
| INP1 | ai_image | Joe 自己摆拍好的多色原图（构图+产品真源） |
| IN3 | ai_image | 色准图①（蓝图里已有这张卡） |
| IN3B | ai_image | 色准图② |
| IN3C | ai_image | 色准图③ |
| GSCENE | ai_image | 换场景成品图，3:4 / 2K / high |

连线顺序（**图槽序，不得乱动**）：`INP1 → IN3 → REQ2 → IN3B → IN3C`
REQ2 是文本卡不占图槽，所以图号 = INP1(图1) / IN3(图2) / IN3B(图3) / IN3C(图4)，
与提示词里的声明一致。

**为什么这条路成立**：它是**编辑**指令，不是生成指令。之前所有 G 卡都在让模型
"重画一条内裤"，所以 logo、颜色、版型每轮都漂；这张卡把产品像素当既成事实保留，
AI 只干它擅长的（换背景、重打光、加投影）。产品真实性从源头保证，不靠提示词去追。

**调参历史（别退回去）**：
- 初版写了"摄影师留下的褶子也得留着"（防重画），结果褶皱全保留，出图像没熨过。
- 现版改为区分**结构性折叠**（腿口翻边/裆部拼接/立体轮廓，保留）和
  **随机表面褶皱**（抚平）。副作用：logo 文字比不抚平时略糊，小尺寸可接受。
