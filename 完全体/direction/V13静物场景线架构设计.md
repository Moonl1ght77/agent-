# V13 静物场景线 · 架构设计

> 2026-07-22 定案并落地。产品线：无模特、无文字的商用静物场景生图线。
> 内裤起步，引擎类目无关（铁律21），任意平铺软织物（袜/内衣/背心等）可复用。
> 真源：`blueprints/workflow_spec_v13.json` + `blueprints/prompts/*_v13.txt`。改提示词只动这里，重跑 build_lxf。

## 一、需求（Joe 拍板）
- 用途：内裤电商详情页/主图，淘宝/1688，比例 **3:4**。
- 形态：**场景里摆着产品**，无模特、无文字。
- 摆法：混合 4 种——平铺主图 / 悬挂 / 折叠 / 面料微距。
- 一款多配色（3-4色不定）：每配色一张白底图，逐色跑。
- 第一优先级：**面料/纹理/花纹不变形**。
- logo：AI 全画，**接受偶尔拼错**（英文小字买家不看/装饰性；在乎准确才后期贴）。

## 二、可行性验证（2026-07-22 探针，独立项目 7da14590）
真源图 = 燕麦灰同色提花针织 + 米白蕾丝边 + 微小手写体logo的女内裤（IMG_0892，硬骨头样本）。
- **场景 AI 生成**（无场景参考图）：优秀，高级静物调性。
- **纹理保真**：R12 恐惧未发生——提花底布未被抹平、蕾丝清晰、颜色准。结论 = **"形似色准"**（面料家族/颜色/蕾丝对，精确纹样会漂，生成式物理天花板）。Joe 判"总体很像"通过。
- **三摆法全通**：平铺✅/折叠✅/悬挂✅（悬挂形变担忧未发生）。
- **logo**：AI 能画出花+珍珠+部分文字，但主品牌词会抽奖拼错（huajingcaihao→huajingcaihxe）。Joe 接受。
- 关键结论：**平铺/摆放=低形变场景**，纹理保真远好于 V12 云皱棉（那是"面料做成成衣穿上身"的高形变场景）。

## 三、架构（无C精简拓扑）
```
需求卡①品类输出 / 需求卡②场景与摆法
IN1 产品白底图（逐配色替换，唯一图槽）
  → A1 产品DNA锁定官（读白底图→面料/纹理/蕾丝/颜色/logo英文描述 + 纹理常见度档）
  → G1平铺主图 / G2悬挂 / G3折叠 / G4微距   （槽1=IN1产品图；A1描述+REQ2场景作文本注入；3:4·2K）
  → QA 四图质检（保真/形变/摆法/幻觉文字/真实感）
运行组：①分析(A1) → ②生图(G1-4,计费) → ③质检(QA)
```
规模：10卡 / 21连 / 3组 / 1媒体占位。

## 四、为什么砍掉 C（对框架 C 惯例的有意偏离，Joe 核准）
- 模特线的 C 核心职责 = 8张同人同景的**跨图一致性**（同一场景句逐字复用、同版型、同鞋）。
- 本线 4 张是**故意不同的摆法+不同场景**，不需要跨图统一；一致性由"同一张产品图进4卡 + A1 同一份描述注入"保证。
- C 在此为空转 → 砍掉省一次 LLM 调用/次。同理砍 P（无C不用切分）、A场景（场景固定写进G卡、无场景输入可分析）、X详情页（Joe没要拼版）、R返修（单卡重跑够，QA报警再加）。
- A1 保留：探针证明"显式点名提花浮雕、别抹平"对保真有用，A1 自动 per 配色生成这句。QA 保留：保真是第一优先级，必须有检查位。

## 五、通用性（铁律21，Joe 要求"可能要通用"）
- A1/G/QA 全部**类目中立**：不写死"燕麦灰提花蕾丝内裤"，只说"reproduce the product in Input Image 1 as locked by PRODUCT_DNA"。具体面料/颜色全从图 + A1 来。
- 换配色 = 换 IN1 白底图重跑；换别的平铺产品 = 同样能跑（A1 字段按实际产品填）。
- 品牌/类目特征只进数据层（需求卡 + A1 字段），规则引擎品牌无关。

## 六、继承的护栏（移植自现有踩坑）
- **R12 纹理诚实分级**：A1 字段8 纹理常见度档（common/uncommon/rare），uncommon 以上预警"形似色准"、核可对照原片。
- **R13 抗AI感**：QA 维度8 真实感/抗塑料感/抗CGI；G卡负向 not plastic/not CGI/not over-sharpened。
- **R14 版本戳**：A1/QA 输出第一行 ENGINE V13.0，QA 核对上游锁卡版本戳。
- **铁律13 禁字**：G卡"no text except product's own logo"；X类拼版本线未做。

## 八、V13.1 白底SKU前置（2026-07-23 公司机，Joe 需求变更）

变更起因：Joe 的实拍图褶皱多、拍在深色布上，手上没有干净白底图 —— 白底 SKU 图本身也要 AI 做，且它同时是交付物（淘宝白底主图）和下游 4 张摆拍图的图源。

```
IN1 产品实拍图（任意背景/有褶皱）
  ├─→ S0 白底SKU图（1:1·2K）— 重绘成纯白底、去褶皱、平铺俯拍 → G1-G4 槽1
  └─→ A1 产品DNA锁定官 ─────────────────────────────────────→ G1-G4 文本注入 + QA
QA 收 IN1(实拍真源) + S0 + G1-4 + A1
```
规模：11卡 / 24连 / 4组。运行组：①S0(核可闸口) → ②A1 → ③G1-4 → ④QA。

两条设计铁则（改这条线时不许动）：
1. **A1 读 IN1 实拍原片，不读 S0 重绘图**。真实纹理只存在于真实照片里；S0 是生成式重绘产物，拿它当真源 = 错纹理被如实锁定并全链复制（V12 云皱棉 GIGO 实锤）。
2. **S0 单独成组做人工核可闸口**。S0 是下游 4 张的唯一图源，S0 错则 4 张全错；单卡重跑 S0 的成本远低于重跑 4 张 2K。

S0 特有的失败模式（QA SKU_CHECK 专项已覆盖，均为"AI 重绘 ≠ 抠图"带来的）：
- 版型被偷改（腰头高度 / 腿口形状 / 缝线走向）
- 浅色被漂白（浅米、浅粉最易被洗成纯白）
- 熨过头 → 塑料/CG 感（软针织应保留自然轻微起伏）
- 抠图边缘毛边 / 白边光晕、凭空多出吊牌衣架装饰

诚实边界（写进 DOC1）：要像素级真实只能真去拍白底或人工抠图；S0 给的是"干净好看且版型基本对"，不是复制。

★ CJK logo 风险（本批内裤实锤输入）：产品 logo 含中文（"树叶趋同粉色，建议更换" + Biological clock + 叶片图标）。AI 画英文小字是抽奖拼错，**画中文基本必糊成假字**，比英文严重一档。A1 字段9 与 QA SKU_CHECK f 项都已强制点名。备选方案 = S0 出图后后期把真 logo 贴回（贴一次，多配色复用）。待 Joe 首轮看糊的程度定。
