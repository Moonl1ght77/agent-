# 通用框架 —— agent 全自动生图工作流概念库

> 这是一个**可裂变的通用框架**：沉淀了已被实测验证的架构理念、硬规则和裂变流程。
> 用途：接到新类目（不同品牌服装/其他商品）时，任何 AI 助手照本框架裂变出一套定制工作流，
> 不需要重新发明架构，也不允许乱发挥。
> 验证来源：V9.1 定制版 8 分镜工作流（首轮实测可用率 7-8/8，超 85% 商用目标）。

## 给 AI 的开工指令（无论你是什么模型，先读这段）

你的任务不是"设计一套新工作流"，而是**执行一次裂变**：把本框架的通用骨架特化成目标类目的定制工作流。

**阅读顺序（强制）：**
1. `blueprints/硬规则铁律.md` —— 先读铁律。每条都有实测翻车史，违反必翻车。
2. `direction/核心理念.md` —— 理解为什么这样设计（DBS 解耦 + 九条架构判断）。
3. `direction/通用骨架.md` —— 通用拓扑：哪些是骨架（不许动），哪些是插槽（按类目定制）。
4. `blueprints/定制问卷.md` —— 拿问卷去问用户，收集齐答案才许动手。
5. `direction/裂变流程.md` —— 按步骤执行裂变，每步有验收标准。
6. `blueprints/节点提示词模板.md` —— 写各节点提示词时照模板骨架填。
7. `solutions/工具层规范.md` —— 写/改构建脚本时的强制要求与交付前验收清单。

**三条总禁令（低模型最容易犯）：**
- ❌ 禁止跳过问卷直接开工。图看不出来的商品属性（松紧/垂感/弹力）必须问出来做成硬指令字段。
- ❌ 禁止"顺手优化"骨架拓扑、槽位协议、抽取器模式、QA 输出结构。这些是验证过的骨架，改之前必须向用户说明理由并获同意。
- ❌ 禁止绕过蓝图直接改平台画布或 .lxf 文件。唯一真源永远是 blueprints。

**成功标准（做到才算完成）：**
- 蓝图（spec + prompts）能被构建器一次性编译通过，核验脚本报告零漂移；
- 用户先跑"分析编译组"能看到人类可读的锁卡，改需求卡即可迭代，不用改提示词；
- 试跑一轮后，QA 报告的可用率 ≥ 目标（商用默认 85%）。

## 参考实现

本仓库 `完全体/` 目录是本框架的第一个落地实例（女装/健身服类目，LumaX Flow 平台）：
- `完全体/direction/架构总设计.md` —— 实例的完整设计决策记录
- `完全体/blueprints/` —— 实例蓝图（spec + 全部提示词），裂变新工作流时的最佳起点
- `完全体/solutions/` —— 构建器与 apply 脚本，新平台可参考其校验逻辑
