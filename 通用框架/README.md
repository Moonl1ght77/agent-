# 通用框架 —— agent 全自动生图工作流概念库（按"三年可用"标准维护）

> 这是一个**可裂变的通用框架**：沉淀了 V9.0→V11.1 共11个版本迭代的全部实测验证。
> 用途：接到新品类/新品牌，任何 AI 助手照本框架半天裂变出一条合格产品线，零串味、不乱发挥。
> 验证来源：三条已落地产品线（V9.2全身生活版 / V10半身生活版 / V11健身棚拍半身版，最新实测 6/8 直接可用+2张定位到确定性根因已修）。

## 给 AI 的开工指令（无论你是什么模型，先读这段）

你的任务不是"设计一套新工作流"，而是**执行一次裂变**：把通用骨架特化成目标品类的定制产品线。

**阅读顺序（强制）：**
1. `direction/防串味总纲.md` —— 四类串味的机制与护栏，最高优先级。
2. `blueprints/硬规则铁律.md` —— 21条铁律，每条都有实测翻车史。
3. `direction/核心理念.md` —— DBS 解耦 + 十一条架构判断 + 商用规则库。
4. `direction/通用骨架.md` —— 骨架（🔒不许动）vs 插槽（🔧按品类定制）。
5. `direction/三年复用规划.md` —— 接单SOP七步 + 大品类预判问题清单P1-P7。
6. `blueprints/定制问卷.md` —— 问卷全答完才许动手。
7. `direction/裂变流程.md` + `blueprints/节点提示词模板.md` + `solutions/工具层规范.md` —— 动手时照着做。

**低模型执行卡（低等级模型必须逐条遵守，抄进你的工作笔记）：**
- 你只允许改四样东西：需求卡默认值、分析官字段值、分镜表行、类目规则段。骨架拓扑/槽位协议/抽取器/QA输出结构/工具代码，一律禁改。
- 每次改动必须走发布闭环四步：`build_lxf.py` → `verify_lxf.py` 零漂移 → `apply_to_app.py` → `verify_canvas.py`。**以脚本结论为准，不以你自己的判断为准。**
- 修任何问题之前，先查 `完全体/direction/回归防线.md` 登记表；修完必须登记（条款+QA项+清单项三点落地）。
- 图案/logo保真短语必须逐元素逐色带数量枚举（铁律19）；小元素先算像素账（铁律20）。
- 拿不准=停下问用户（铁律14）。禁止猜、禁止编字段、禁止跳过问卷、禁止在旧画布项目上测新版本。
- run 计费：批量运行前必须用户确认；先组②验锁卡再花钱跑组③。

**成功标准：** 构建器一次编译通过+核验零漂移；用户改需求卡即可迭代；QA 可用率 ≥ 85%（差1-2张用组⑤单卡返修补齐）。

## 参考实现（三条产品线）

`完全体/` 是本框架的落地实例库：
- `blueprints/workflow_spec.json`（V9.2全身生活版；`--variant half`=V10半身生活版）
- `blueprints/workflow_spec_v11.json`（V11健身棚拍半身版：6输入+4需求卡+S2细节节点+2K）
- `solutions/` 工具链5件套（build/apply/verify_lxf/verify_canvas/make_detail_crop，全部支持 --spec/--variant）
- `direction/回归防线.md`（R1-R11 已修问题登记表+产品线登记表+发布闭环）——**改任何提示词前必查**
