> 目标：接到任何新品类/新品牌，都能在半天内裂变出一条合格产品线，且与既有产品线零串味。

## 二、新品牌/新品类接单 SOP（7步，半天级）

1. **问卷**（`通用框架/blueprints/定制问卷.md`，22问）：重点=硬指令字段清单（图看不出来的属性）+类目主图规范+卖点位置。禁止跳过（铁律18：输入自洽）。
2. **素材收集规范**：向商家要——模特图（必须带体型）、主商品正面白底原片（越高清越好，细节裁剪靠它）、背面图（背部有卖点必要）、场景/背景参考、姿势参考、穿搭参考。归档到 `素材X/品牌名-用途/`。
3. **选基线**：生活场景类→V9.2/V10；棚拍目录类→V11；比对差异后复制最近似 spec 为 `workflow_spec_<品牌线>.json`，新建对应 REQ/A3（必要时A1/A2/C/QA）的 `_<线名>` 后缀提示词文件。品牌特征只进数据层（铁律21）。
4. **裂变改动清单**（按序）：REQ卡默认值 → A1类目字段（新物理属性四级贯通）→ 分镜表（类目主图规范+卖点位）→ C类目规则段 → QA对应维度与硬失败。骨架拓扑/抽取器/QA输出结构不动。
5. **构建+核验**：build --spec → verify_lxf --spec 零漂移 → apply 新建画布项目（项目名带版本）→ verify_canvas 回读。
6. **小成本试跑**：组②看锁卡（便宜可反复）→ 达标才跑组③（计费，2K更贵）→ 组④QA（1次调用，比人眼快且直接喂返修）→ 个别失败走组⑤单卡返修。
7. **收工登记**：回归防线产品线登记表+记忆三份+双仓库推送+【记忆存档回执】。

## 三、大品类扩张预判问题清单（越早知道越省钱）

### P1 批量SKU（如整季15款/35SKU连跑）
现状：apply 接口不能写图（imageUrl不在白名单），换款只能画布手动换输入图。
**第一步已实现**：`season_inventory.py <整季根目录> --crop` —— 纯Python扫描每款文件夹，
分类"全身平铺/细节特写"（白边占比，实测16款零错）、自动分线"彩色刺绣款/文字纯色款"
（色相簇数+尺寸闸，实测~90%准，错例经裁图1秒复核可抓）、自动裁品牌logo细节图。
产出 `_季度清单.txt/json`：每款的输入候选清单，把"手动翻225张图挑输入"变成一条命令+人工复核。
对策：同一项目逐款按清单换图跑（锁卡会自动重析）；产出按 `款号/G1-G8+QA报告` 归档。
若平台未来开放素材API再做全自动循环——**别现在造轮子**（规则5）。
风险：①连跑时忘换某张输入图=内容串味，清单化每款跑前核对；②分拣器对金属五金/彩色滚边
会个别误判品类（实锤：抽绳裤银头被判刺绣），故品类是"建议+裁图复核"，绝不让代码authoritative
（真源优先+人工检查点）。

### P2 新品类的"图看不出来"属性
裤装=腿型裆部走向；羽绒=充绒蓬度；泳装=水感反光；丝绸=高光；鞋=足姿视角。
对策：问卷A2题必答→做成需求卡硬指令字段→四级贯通（REQ→A→C→QA），模板照抄版型链路（R2/R3）。

### P3 卖点是大段文字/复杂印花
AI生成文字必乱码（铁律13），大logo印花在2K下可画但逐字风险高。
对策：印花走 逐字锁+枚举令+特写位+2K+真源细节图 五件套；超过几个单词的大段文字直接后期贴图，不要让生图硬画。

### P4 低等级模型执行（最大风险）
低模型的典型翻车：乱改骨架拓扑、跳过问卷、编造字段、概括性复述锁卡、不跑核验直接交付。
对策=**低模型执行卡**（见 README 开工指令区，逐条抄给它）：
- 只允许改：需求卡默认值、A官字段值、分镜表行、类目规则段。其余一律禁改。
- 每次改动必须走：build → verify零漂移 → apply → verify_canvas 四步，**用脚本结论代替自我判断**。
- 修任何问题前先查回归防线登记表；修完必须登记。
- 拿不准=停下问用户，禁止猜（铁律14）。
- 交付标准=`通用框架/solutions/工具层规范.md` 验收清单全勾。

### P5 平台/模型升级
平台协议变：先 GET /ping+/manifest 探测，graph-schema 对拍，新字段单卡试（铁律15）。
生图模型换代（gpt-image-2→新模型）：参考图槽位上限、前缀习惯、分辨率档位都可能变——先单卡试跑一张验证前缀声明是否仍被遵守，再全量；英文提示词层与锁卡层不需要动。
编排模型换代：锁卡字段结构是合同，换模型后先跑组②比对锁卡格式是否仍守约。

### P6 成本经济学（品类越大越重要）
分层花钱：组②LLM调用（分毫级，可无限迭代）→ 组④QA（1次调用，替代人眼筛图）→ 组⑤单卡返修（1-2张）→ 组③全量8张2K（大头）。
铁律：锁卡不过关不跑③；个别失败用⑤不重跑③；系统性失败回蓝图（改完只重跑②+失败卡）。
经验值：V11首跑 6/8可用+2张单卡返修 ≈ 最优成本路径。

### P7 记忆规模化
产品线多了以后：项目记忆仍是三份（不按产品线拆，产品线状态进"回归防线登记表"）；跨项目经验只走知识库晋升制（防记忆串味第四类）。每季度做一次"记忆检查"（INDEX.md清单）。

## 四、三年演进路线（按需触发，不预开发）
- **近期**（当前品牌35款）：V11.1 跑完8/8 → 批量SKU连跑 → S2独立分组（已登记待办）。
- **中期**（新品类进来）：按SOP裂变；每成功一条线，把二次验证的打法晋升知识库；通用框架此时即完成"第二次验证"，整体晋升40-知识库指路。
- **远期**（工作流数量>5条线）：考虑 blueprints 拆库（每产品线一个子目录）；apply 网关若开放素材API做批量自动化；QA自动触发返修（平台支持条件执行时）。
- **永远不做**：预开发没有单子的品类模板、把品牌特征写进规则引擎、绕过发布闭环的"快捷改动"。
