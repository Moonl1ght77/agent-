# 通用骨架 —— 平台无关拓扑：骨架 vs 插槽

> Direction 层。定义什么永远不变（骨架）、什么按类目定制（插槽）。
> 低模型注意：本文标 🔒 的部分不许改；标 🔧 的部分按《定制问卷》答案填。

## 一、通用拓扑（🔒 骨架）

```
[需求卡① 品类与输出]────┬──────────────────┐
[需求卡② 硬指令字段集]──┼──────────────┐   │
                        │              ↓   ↓
IN_模特 ────────────────┼─────→ A2 人物锁 ──┐
IN_商品 ─┬──────────────┼→ A1 商品锁 ──────┼→ C 总控编译器 → P1..PN 抽取器 → G1..GN 生图 → QA 质检
IN_细节 ─┤              │                  │        （N = 分镜数）              │
IN_场景原图 → S 场景清洗┼→ A3 场景+分镜锁 ─┘                                   ↓
IN_姿势 ────────────────┘                                             R 返修编译 → RG 返修生图
IN_搭配单品(可选) ────────────────────────────→ 直连生图卡槽位（所见即所得）
G成品 + IN_商品 → X 组合输出（四宫格/详情页首屏等）
```

**运行组（🔒 六组结构不变）**：
①预处理（清洗类节点）→ ②分析与编译（A×3 + C + P×N）→ ③生图（G×N）→ ④质检（QA）
→ ⑤返修支线（R+RG，按需）→ ⑥组合输出（X 类）

**层内并发**：A1/A2/A3 并行、P1..PN 并行、G1..GN 并行（平台自动分层调度）。

## 二、固定节点职责（🔒 职责不变，🔧 内容按类目填）

| 节点 | 职责 | 输出 |
|---|---|---|
| S 清洗 | 场景原图去人/去水印/去杂物 | 净化场景参考图 |
| A1 商品锁定官 | 读商品图+细节图+需求卡，输出商品DNA锁卡 | 结构化字段：品类廓形/主色+色号/面料物理/**版型上身状态（硬指令覆盖）**/图案Logo几何/结构细节/五金辅料/风险提示 |
| A2 人物锁定官 | 读模特图+需求卡，输出人物DNA锁卡 | 脸型五官/肤色深浅/发型/身材（如实）/造型计划 |
| A3 场景分镜官 | 读净化场景图+需求卡，输出场景DNA+分镜表 | 场景光影色温 + SHOT_PLAN_N（每镜：机位/景别/动作/用途） |
| C 总控编译器 | 三张锁卡+需求卡 → N 组英文正/负提示词 | 每组含：槽位角色前缀 + 商品锁 + 人物锁 + 场景锁 + 该镜构图 + 硬指令短语 + 负向词 |
| P 抽取器 | 原样抽取第 X 组，一字不改 | 单组提示词 |
| G 生图卡 | 按固定槽位挂参考图 + 单组提示词 | 成品图 |
| QA 质检 | **按内容认图**，对照锁卡与提示词逐图打分 | 结构化报告：PASS_SUMMARY/构图核对/评分表/硬失败清单/最佳图/最弱图/返修简报 |
| R 返修编译 | 读 QA 返修简报 → 一组英文返修提示词 | 给 RG 用 |
| RG 返修生图 | 与原生图卡同槽位协议，单卡重跑 | 替换图 |
| X 组合输出 | 成品+商品图 → 详情页首屏/四宫格细节等 | 营销素材（禁文字占位） |

## 三、参考图槽位协议（🔒 协议机制不变，🔧 槽位语义按类目定）

- 每张生图卡的**连线顺序 = 槽位顺序 = 提示词前缀声明顺序**，三者由构建器程序保证一致。
- 挂几张图就声明几张，声明与实际严格一致（历史坑：声明6张实挂5张，姿势图只存在于文字里）。
- 服装类目验证过的默认协议：槽1模特 / 槽2商品 / 槽3细节 / 槽4搭配单品 / 槽5净化场景 / 槽6姿势（仅姿势参考镜挂）。
- 新类目按问卷重定槽位语义，但"固定槽位+角色前缀"机制不变。

## 四、分镜表（🔧 纯插槽，按类目主图规范定制）

分镜数 N 默认 8。服装电商（淘宝主图标准，已验证）：
前 4 镜 = 版型标准位（正面全身 / 背面全身 / 正面近景 / 45°侧身），后 4 镜 = 同场景生活位。
其他类目按该类目详情页刚需重排（例：箱包=容量展示位+背身比例位；家居=场景氛围位+尺寸对比位）。
**每类目必含**：主图候选位、细节信任位、背面/反面结构位（详情页刚需，容易被漏）。

## 五、需求卡（🔧 字段按类目定，机制 🔒）

- 需求卡① 品类与输出：品类、目标平台、输出比例/数量、风格基调。
- 需求卡② 硬指令字段集：**所有"图看不出来"的属性**（服装：版型上身松紧；按问卷补充类目特有项）。
  每个硬指令字段必须四级贯通：需求卡 → A1/A2 对应字段强制覆盖 → C 每组提示词注入 → QA 专项+硬失败。
- 用户只改需求卡就能迭代，不许要求用户改提示词。
