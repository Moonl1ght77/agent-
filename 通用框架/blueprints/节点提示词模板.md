# 节点提示词模板 —— Agent 八要素 + 各节点类型骨架

> Blueprints 层。写新节点提示词时照骨架填空；完整活例见 `完全体/blueprints/prompts/*.txt`。
> 语言分工：分析类中文指令+英文值 / 编译器纯英文 / 抽取器中文一句话 / QA·返修中文报告。

## 〇、Agent 八要素（所有 LLM 节点通用骨架）

```
1. 名称与角色：你是本工作流的「XXX」节点。
2. 输入清单：你收到的输入是……（逐项列明每张图/每段文本是什么）
3. 禁读清单：不引用历史结果 / 不读XX / 只审核本轮输入。
4. 目标：一句话说清产出什么、给谁用。
5. 输出字段结构：固定字段名+顺序（下游按结构解析，不许自由发挥）。
6. 不确定处理：拿不准写 UNCERTAIN，禁止猜测。
7. 下游声明：你的输出将被「XXX」节点使用。
8. 边界：不生成图片 / 不修图 / 不输出多余解释。
```

## 一、分析 Agent（A1 商品锁 / A2 人物锁 / A3 场景分镜锁）

要点：
- 中文指令；输出**中文字段名 + 英文值**（值直接进英文生图提示词）。
- 字段固定编号（字段1…字段k），每字段说明"从哪张图读+读什么"。
- **硬指令字段特殊写法**（以版型为例）：`字段4 版型上身状态：以需求卡②的「版型上身要求」为准，强制覆盖你从平铺图得出的推断；需求卡为空才允许视觉推断。`
- 风险提示字段兜底：把拿不准的、图与需求矛盾的写进"风险提示"，给人看。

## 二、总控编译器 C

要点（纯英文提示词，结构骨架）：
- 输入映射：三张锁卡+需求卡各是什么。
- 槽位角色前缀模板：每组以 "Input Image 1 is the model… Input Image 2 is the garment…" 开头，前缀版本数与实挂图数严格一致（铁律8）。
- 分镜导演规则：逐镜定义构图/景别/动作/表情/朝向，N 组互不重复。
- 硬指令执行法（Fit Execution Law 模式）：每组必须包含硬指令正向短语；对应负向词**常驻每组**（不是只写一次）。
- 十条商用规则段（见核心理念）+ 类目特有规则段。
- 跨图一致性条款：N 张同人同商品同松紧同搭配同场景。
- 输出结构：`=== SET X (正向) === / === SET X NEGATIVE ===` 固定分隔，供抽取器切分。

## 三、抽取器 P

全文即一句话（不许扩写）：
```
从上游文本中原样取出「SET X」的正向提示词与负向提示词，一字不改，不添加任何解释。
```

## 四、QA 质检

结构骨架（完整活例：完全体/blueprints/prompts/QA_质检八图.txt）：
1. **认图规则前置**（铁律3）：两步内容认图——先分参考图/成品图，再按构图特征匹配分镜；缺失/重复=构图失败；输出前自检"若在按输入顺序编号，作废重来"。
2. 工作流目标一段：风格基调（真实感标准）。
3. 检查维度：构图执行 / 商品保真（以商品参考图为唯一事实源）/ **硬指令专项**（如版型合身度）/ Logo图案几何 / 人脸身份肤色 / 身材真实 / 场景一致 / 曝光双保护 / 搭配边界 / 跨图一致性。
4. 硬失败标准：逐条可判定（与硬指令字段一一对应，含"松紧与锁定相反=硬失败"模式）。
5. 评分口径：9-10 主图候选 / 7-8.9 可用 / 5-6.9 风险 / 0-4.9 硬失败。
6. 固定输出结构：PASS_SUMMARY / STRUCTURE_CHECK / SCORE_TABLE（含"构图识别"一句话描述列，供人工对照）/ HARD_FAILS_BY_REF / BEST_IMAGE / WEAKEST_IMAGE / REPAIR_BRIEF。

## 五、返修编译 R

- 输入：QA 报告（重点 WEAKEST_IMAGE + REPAIR_BRIEF）+ 原提示词组。
- 输出：一组可直接使用的英文返修提示词（保留原构图定义，叠加问题修正短语）。
- 边界：只处理 QA 点名的那张图；不生成图片。

## 六、需求卡（text 卡，用户填写）

- 需求卡①：品类 / 目标平台 / 输出比例数量 / 风格基调。
- 需求卡②：硬指令字段集，**每个字段给默认值**（用户不填也能跑出品牌默认效果）。
- 卡内注释写清"改这里即可，不用改任何提示词"。

## 七、组合输出 X（详情页/四宫格）

- 结构化构图指令（版式区块逐块描述），**禁止文字占位**（铁律13），留白给后期排版。
