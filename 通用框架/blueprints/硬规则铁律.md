# 硬规则铁律 —— 15 条，每条带违反后果与验证方法

> Blueprints 层。这些不是建议，是实测翻车换来的铁律。任何模型执行本框架时逐条遵守；
> 想违反任何一条，必须先向用户说明理由并获得明确同意。

## 铁律 1：连线顺序 = 参考图槽位顺序 = 提示词前缀声明顺序
- **违反后果**：生图模型把商品当场景、把模特当细节，一致性崩塌且随机劣化（素材A实锤：靠大模型语义自愈"能跑"，但就是隐性质量地板）。
- **验证**：构建器程序校验三者一致；人工抽查任一生图卡的入边序 vs "Input Image N is ..." 声明序。

## 铁律 2：蓝图是唯一真源
- 改提示词只改 `prompts/*.txt`，改结构只改 `workflow_spec.json`，然后重跑构建器。禁止直接改 .lxf / 画布后不回写。
- **违反后果**：蓝图与产物漂移，下次重建静默覆盖你的画布改动，问题"复活"。
- **验证**：核验脚本（蓝图推导期望态 vs 产物实际态）报告零漂移才算完成。

## 铁律 3：多图 LLM 节点一律按内容认图，禁按序号认图
- 并发生成导致喂图顺序随机。QA 等一次收多张图的节点必须两步认图：先按视觉特征区分参考图/成品图，再按构图特征匹配到分镜；匹配不上/重复匹配本身就是构图失败。
- **违反后果**：V9.1 首轮实锤——6 个"硬失败"全是序号错位的误报，QA 报告整体作废。
- **验证**：QA 提示词含"禁止按输入顺序认图"+输出前自检条款；报告的构图识别列附一句话描述供人工对照。

## 铁律 4：图看不出来的属性必须四级贯通
- 松紧/垂感/弹力/厚薄等视觉推断不出的属性：需求卡显式字段 → 分析Agent强制覆盖视觉推断 → 编译器每组正向短语+负向词常驻 → QA 专项检查+硬失败条款。
- **违反后果**：实锤——需求写"紧身"仍出宽松图，因为分析节点只看了平铺图。
- **验证**：沿链路搜硬指令关键词，四级都出现才算贯通；试跑组②检查锁卡字段值。

## 铁律 5：平台禁环，反馈做返修支线
- QA→生图不许连回路；做成 QA→R 返修编译→RG 返修生图的向下支线+单卡重跑。
- **违反后果**：平台拓扑排序直接报错，整个工作流跑不起来。
- **验证**：构建器内置无环校验（拓扑排序计数==节点数）。

## 铁律 6：run 计费，先廉后贵
- 组间人工检查点：先跑分析编译（便宜）人工验锁卡，锁卡过关才跑生图（贵）。批量 run 前必须用户确认。
- **违反后果**：花钱生出 8 张锁卡就能预见的废图。
- **验证**：裂变流程第 4 步顺序执行记录。

## 铁律 7：抽取器只抽取，禁改写
- P 节点指令固定为"原样取出第 X 组，一字不改"（中文一句话最不易被误执行）。
- **违反后果**：各分镜自行改写 → N 张图风格漂移，跨图一致性失守。
- **验证**：P 输出与 C 输出对应组 diff 为空。

## 铁律 8：声明与实际严格一致
- 挂几张参考图就声明几张；某镜多挂姿势图就单独用多一张的前缀版本。
- **违反后果**：素材A实锤——声明6张实挂5张，"姿势参考"只存在于文字里，从未生效。
- **验证**：每张生图卡入边数 == 其提示词前缀声明数。

## 铁律 9：Agent 八要素提示词骨架
- 每个 LLM 节点提示词必须含：名称/角色、输入清单、禁读清单、目标、输出字段结构、不确定处理（写 UNCERTAIN 禁猜测）、下游是谁、输出语言。
- **违反后果**：自由发挥的提示词=不可预测的输出结构，下游抽取/质检解析失败。
- **验证**：对照《节点提示词模板》逐要素检查。

## 铁律 10：结构性改动 apply 成新项目，禁原地升级
- apply 不隐式删连线；删卡重连会把新连线追加到队尾 → 槽位错位。
- **违反后果**：静默的槽位错位，图面上表现为"莫名其妙不像了"。
- **验证**：结构变更后的项目必须是新建画布；老画布只做 content-only 热更新。

## 铁律 11：content-only 单卡热更新是安全的
- 只发一张卡的 ref+type+data.content 可精准更新提示词，不碰连线、不覆盖用户其他卡。结构不变时优先单卡 apply，不全量重发。
- **验证**：热更新后拉取画布确认其余卡 updated_at 未变。

## 铁律 12：蓝图自包含，禁机器特定路径
- 占位图等资产入库 blueprints/assets/；工具只用相对仓库路径；平台安装路径做成文件头部配置常量。
- **违反后果**：实锤——构建器写死公司机临时目录，换电脑无法重建。
- **验证**：新机器 clone 后构建器直接跑通。

## 铁律 13：AI 生成图内文字必乱码
- 详情页/海报类输出禁文字占位，构图留白，文字后期排版加。
- **验证**：X 类节点提示词含"禁止生成任何文字"条款。

## 铁律 14：不确定就问，禁猜测
- 分析 Agent 拿不准的字段写 UNCERTAIN；裂变执行者拿不准的需求回问用户。商用工作流里一个猜错的字段会被复制到 N 张图。
- **验证**：锁卡里出现 UNCERTAIN 时，编译器规则是"跳过该属性"而非编造。

## 铁律 15：平台接口白名单意识
- apply 的 data 字段超出白名单会**整体拒绝**（非静默丢弃）；导出格式(width/height/data字符串) ≠ apply格式(w/h/data对象)；apply 不含 imageUrl（搭好的图输入卡需手动放图）；加密卡只能连线使用不可读写。
- **验证**：接口调用前 GET /ping 与 /manifest 确认登录态与能力；新字段先单卡试。
